Windows 对象管理

1 内核对象

对象分为内核对象、用户对象。用户对象主要用于 GUI 下（没有对象头）。

  • 内核对象：包含对象头、对象体。
    • 无名对象，其他进程不能通过对象名打开对象，对象不可以共享。不命名的对象一般意义不大。
    • 命名对象，可以通过对象名与其他进程共享内核对象。
  • 用户对象：不具有对象头，参见《GUI Subsystem 用户对象》。

1.1 对象目录

在系统中，通过对象目录对命名对象进行组织管理，对象目录是树状结构而非网状结构。

  1. 根节点： 目录对象（_OBJECT_DIRECTORY）。根节点对象名为\。
  2. 中间节点：目录对象（_OBJECT_DIRECTORY）、符号链接对象（_OBJECT_SYMBOLIC_LINK）。
  3. 叶节点：普通内核对象、目录对象（_OBJECT_DIRECTORY）。

说明：根节点是一个数据目录类型，节点名为 “\”，其为 37 个元素的哈希数组，每个元素串起一个单链表队列，同一个链表上的所有节点对象名的 Hash 值相同。

目录对象：可以是根节点、中间节点、叶节点。

主要关注第一个成员，可以看到它是一个含有 37 = NUMBER_HASH_BUCKETS 个成员的指针数组，每个成员都指向 _OBJECT_DIRECTORY_ENTRY 结构。

  • ChainLink：指向下一个 _OBJECT_DIRECTORY_ENTRY，是个单向链表。同一条链表上的所有节点对象名的 Hash 值相同。
  • Object：指向该 Entry 所管理的那个对象，比如 EPROCESS，ETHREAD 等等。
  • HashValue：该 Entry 的 Hash 值

注意：目录项 _OBJECT_DIRECTORY_ENTRY 并非内核对象，但是每一个内核对象插入到对象目录树时，必须有目录项来进行承接才行。

节点遍历：

如果从根节点或某个中间节点开始遍历某个对象，在路径上的每个节点前加 “\” 进行分隔，就形成一个路径名。

  • 绝对路径：根节点\中间节点\叶节点。
  • 相对路径：中间节点\叶节点。

查看/遍历内核对象主要有三种方法：

  1. 通过 WinObj.exe 工具（以管理员权限运行）。

  3. 通过代码遍历。
