0 句柄表

句柄表分为：进程句柄表（包括内核句柄表）、全局句柄表。区别如下：

类型 包含句柄类型 ObjectPointerBits 指向内容 句柄特征
进程句柄表 1、该进程创建和打开的所有线程句柄。
2、该进程创建的子进程句柄。
3、该进程创建和打开的其他内核对象句柄。		 对象头 _OBJECT_HEADER 句柄值 >= 0，且 $Handle ≡ 0(mod\ 4)$
0：在进程结束时使用。
-1：表示当前进程。
-2：表示当前线程。
除进程、线程句柄外，其他内核对象句柄值不是全局唯一的。
内核句柄表 System 进程的句柄表。 对象头 _OBJECT_HEADER 句柄值 <= 0，为负数。且 $Handle ≡ 0(mod\ 4)$
除进程、线程句柄外，其他内核对象句柄值不是全局唯一的。
全局句柄表 1、所有用户模式、内核模式进程句柄。
2、所有用户模式、内核模式线程句柄。		 对象体 _OBJECT_BODY 句柄值 >= 0，且 $Handle ≡ 0(mod\ 4)$。
只有进程和线程句柄。每个句柄值全局唯一。

句柄及句柄表的一些属性：

  1. 句柄是可以重复使用的，如一个线程终止之后，系统可以将该句柄值分配给其他新创建句柄。
  2. 句柄表是分级的，最多分为三级。为了节省内存，句柄表初始时只有一级，当句柄增多后会动态调整为二级或三级句柄表。
  3. 进程句柄上限：使用 24 个有效位，句柄上限为 $2^{24} - 1 == 16,777,215$。

1 进程句柄表

1.1 普通进程句柄表

每个进程的 EPROCESS.ObjectTable 成员指向当前进程的句柄表（_HANDLE_TABLE）。句柄表 HANDLE_TABLE.TableCode 成员需要仔细进行解析。

每个进程的句柄表最多可扩展为三层，初始时为了节约内存空间，只有一层。当进程的引用的句柄增加到一层句柄表无法容纳时（大于 256 个），可根据句柄数量动态扩展至二层或三层。

句柄表 HANDLE_TABLE 结构如下：

主要成员 解释
TableCode 主要有两个作用：
1、指示当前进程句柄表的层数（layers）：layers = TableCode & 0x7。
layers = 0：句柄表为一层结构，TableCode 指向 _HANDLE_TABLE_ENTRY。
layers = 1：句柄表为二层结构，TableCode 不直接指向 _HANDLE_TABLE_ENTRY。
layers = 2：句柄表为三层结构，TableCode 不直接指向 _HANDLE_TABLE_ENTRY。
2、TableCode 是一个指针，如下。
layers = 0：TableCode 指向 _HANDLE_TABLE_ENTRY。
layers = 1：TableCode & ～0x7 指向第二级表，二级表存储的地址都指向 _HANDLE_TABLE_ENTRY 条目。
layers = 2：TableCode & ～0x7 指向第二级表，二级表存储的地址指向第三级表， 三级表存储的地址都指向 _HANDLE_TABLE_ENTRY 条目。
QuotaProcess 指向句柄表所属的进程。
HandleTableList 双向链表，指向其他进程的句柄表，可用于遍历其他句柄表。为 _HANDLE_TABLE 结构。

借鉴 《Windows Internals 7》 将最外层句柄表叫做 Subhandle table，它为 _HANDLE_TABLE_ENTRY 结构。

解释 OBJECT_HEADER.PointerCount 和 HANDLE_TABLE_ENTRY.RefCnt 的关系：

$OBJECT\_HEADER.PointerCount = 该进程句柄个数 + ObReferenceObject函数引用次数 + HANDLE\_TABLE\_ENTRY.RefCnt对该句柄的缓存个数$

对该公式的解释：

  1. 一个内核对象对应多个句柄（该进程句柄个数），每一个句柄都可能被引用多次（HANDLE_TABLE_ENTRY.RefCnt对该句柄的缓存个数）。

  2. 通过 ObReferenceObject 函数可以增加内核对象的引用计数。

  3. 句柄条目 HANDLE_TABLE_ENTRY.RefCnt 表示当前这个句柄值被引用的次数。

    举例，某个进程对象 _EPROCESS 对应的三个句柄值为 240、168、1920；前两个句柄值的引用次数 HANDLE_TABLE_ENTRY.RefCnt == 20，最后一个句柄值的引用次数为 15；通过 ObReferenceObject 函数引用的次数为 3 次。则该进程对象 OBJECT_HEADER.PointerCount = 3 + 20*2 + 15 + 3 = 61 = 0x3D。

ObjectPointerBits 成员是在 Windows 8 新增的成员，用它的值通过计算后来指向该句柄对应的内核对象，不同版本计算方法不同。本文关注 Windows 10 下的计算方法。

在 Windows 10 下该成员为 44 bits。

计算方法一如下：

$$Object\_Header = (HANDLE\_TABLE\_ENTRY.ObjectPointerBits << 4)\ |\ 0xFFFF0000’00000000$$

计算方法二如下：

$$Point = (PVOID)\&HANDLE\_TABLE\_ENTRY;$$

$$Object\_Header = ((Point >> 16)\ \&\ 0xFFFFFFFF’FFFFFFF0)\ |\ 0xFFFF0000’00000000;$$

2 全局句柄表

全局句柄表由全局指针 nt!PspCidTable 指向 _HANDLE_TABLE。全局句柄表有以下几点特征：

  1. 全局句柄表中，只有进程（_EPROCESS）、线程（_ETHREAD）对象的句柄。
  2. 全局句柄表 _HANDLE_TABLE_ENTRY.ObjectPointerBits 经过计算后直接指向进程/线程对象体，而不是对象头。
  3. PspCidTable 中存放的对象是系统中所有的进程线程对象，其索引就是 PID 和 TID。
