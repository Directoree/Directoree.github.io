Chapter 11 Memory Cache Control

The processor allows any area of system memory to be cached in the L1, L2, and L3 caches. In individual pages or regions of system memory, it allows the type of caching (also called memory type) to be specified (see Section 11.5) . Memory types currently defined for the Intel 64 and IA-32 architectures are (see Table 11-2)：

cache line fill（缓存行填充）：When the processor recognizes that an operand being read from memory is cacheable, the processor reads an entire cache line into the appropriate cache (L1, L2, L3, or all). This operation is called a cache line fill.

cache hit（缓存命中）：If the memory location containing that operand is still cached the next time the processor attempts to access the operand, the processor can read the operand from the cache instead of going back to memory. This operation is called a cache hit.如果下次处理器尝试访问该操作数时，包含该操作数的内存位置仍被缓存，则处理器可以从缓存中读取该操作数，而不是返回内存。 此操作称为缓存命中。

write hit（写入命中）：When the processor attempts to write an operand to a cacheable area of memory, it first checks if a cache line for that memory location exists in the cache. If a valid cache line does exist, the processor (depending on the write policy currently in force) can write the operand into the cache instead of writing it out to system memory. This operation is called a write hit.当处理器尝试将操作数写入内存的可缓存区域时，它首先检查缓存中是否存在该内存位置的缓存行。 如果确实存在有效的缓存行，则处理器（取决于当前有效的写入策略）可以将操作数写入缓存，而不是将其写出到系统内存。 此操作称为写入命中。 If a write misses the cache (that is, a valid cache line is not present for area Of memory being written to)， the processor performs a cache line fill, write allocation. Then it writes the operand into the cache line and (depending on the write policy currently in force) can also write it out to memory. If the operand is to be written out to memory, it is written first into the store buffer, and then written from the store buffer to memory when the system bus is available. (Note that for the Pentium processor, write misses do not result in a cache line fil; they always result in a write to memory. For this processor, only read misses result in cache line fills.)如果写入未命中缓存（即，对于正在写入的内存区域，不存在有效的缓存行），处理器执行缓存行填充、写入分配。 然后它将操作数写入缓存行并且（取决于当前有效的写入策略）也可以将其写出到内存中。 如果要将操作数写出到内存，则首先将其写入存储缓冲区，然后在系统总线可用时从存储缓冲区写入内存。 （请注意，对于奔腾处理器，写入未命中不会导致缓存行填充；它们总是导致写入内存。对于此处理器，只有读取未命中会导致缓存行填充。）

处理器允许将系统内存的任何区域缓存在 L1、L2 和 L3 缓存中。 在系统内存的各个页面或区域中，它允许指定缓存类型（也称为内存类型）（参见第 11.5 节）。 目前为Intel 64和IA-32架构定义的内存类型有（见表11-2）：

  • Strong Uncacheable (UC)：不缓存系统内存。所有读取和写入都出现在系统总线上，并按程序顺序执行，无需重新排序。 不会进行推测性内存访问、页表遍历或推测性分支目标的预取。 这种类型的缓存控制对于内存映射 I/O 设备很有用。 当与普通 RAM 一起使用时，它会大大降低处理器性能。

  • Write Combining (WC)：系统内存位置未缓存（与不可缓存的内存一样）并且处理器的总线一致性协议不强制执行一致性。 允许推测性读取。 写入可能会延迟并在写入组合缓冲区（WC 缓冲区，WC buffer）中组合以减少内存访问。 如果 WC 缓冲区被部分填满，写入可能会延迟到下一次序列化事件发生时； 例如 SFENCE 或 MFENCE 指令、CPUID 或其他序列化指令、对未缓存内存的读取或写入、中断发生或 LOCK 指令（包括带有 XACQUIRE 或 XRELEASE 前缀的指令）的执行。 此外，执行 XEND 指令（以结束事务区域）会在逐出在事务区域内执行的任何写入之前逐出在相应执行 XBEGIN 指令（以开始事务区域）之前缓冲的任何写入。

    写入组合缓冲区，write combining buffer（WC buffer）：该缓冲区与内部 L1、L2 和 L3 缓存以及存储缓冲区（Store buffer）分开。写入数据时，虽然数据不会缓存到告诉缓存上，但是数据会被写入到 WC buffer，等 WC buffer 写满之后才会将 WC buffer 的数据写入到内存中。

  • Write-through (WT) ：对系统内存的写入和读取都被缓存。 读取来自缓存命中的缓存行； 读取未命中导致缓存填充。 允许推测性读取。 所有写入都写入缓存行（如果可能）并写入系统内存。 写入内存时，永远不会填充无效的缓存行，而有效的缓存行要么被填充要么无效。 允许写入组合。 这种类型的缓存控制适用于帧缓冲区，或者当系统总线上有访问系统内存但不执行内存访问窥探的设备时。 它强制处理器中的缓存和系统内存之间的一致性。

  • Write-back (WB)：对系统内存的写入和读取都被缓存。 写入数据时会写入缓存行但不会立即写到系统内存，修改后的高速缓存行稍后写入系统内存。 例如在已满的缓存中分配新的缓存行时会将缓存内容写入内存。 这种类型的缓存控制提供了最好的性能，但它要求所有在系统总线上访问系统内存的设备都能够监听内存访问，以确保系统内存和缓存的一致性。

  • Write protected (WP)：读数据时与缓存有关，写数据时不会缓存。
