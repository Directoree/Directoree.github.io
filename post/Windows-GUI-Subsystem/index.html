    1 失败的分析-Windows 10 PsEstablishWin32Callouts

    Win32k.sys 在初始化阶段通过 win32k!SysEntryPsEstablishWin32Callouts 调用 nt!PsEstablishWin32Callouts 函数向内核注册一组出调（callout ） 函数，参考 Geoff Chappell -Software Analyst 在 Windows 10 中数据结构 WIN32_CALLOUTS_OPERATION 的成员，如 Win32ProcessCallout、Win32PowerEventCallout 为出调函数。该结构在 Windows XP 中为 WIN32_CALLOUTS_FPNS。

    nt!PsEstablishWin32Callouts

    nt!PsEstablishWin32Callouts 函数：

    nt!ExAllocateCallBack

    在函数 ExAllocateCallBack 中：

    该函数就是申请了一个 EX_CALLBACK_ROUTINE_BLOCK 结构，并给成员赋值，源码和 XP 中是一样的。

    EX_CALLBACK_ROUTINE_BLOCK 结构参考 ReactOS 0.4.15：

    nt!ExCompareExchangeCallBack

    此函数分配、删除或交换低开销回调函数。该函数只是对当前的快速引用数量上面申请内存 建立的RundownProtect 进行处理。

    最后 mov _PsWin32CalloutsEstablished, 1，表示出调函数已经完成注册。但是实际并没有给出出调函数。这里猜测注册的实现是在 win32k.sys 中完成的。大概率是在参数 win32k!W32CalloutDispatchThunk 和 win32kbase!W32CalloutDispatch 函数有关。
