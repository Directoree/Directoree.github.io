1.2 使用虚拟地址遍历页表

实际在代码中，由于分页模式的存在，我们无法直接操作物理地址。但是 Windows 会将对应的页表结构映射到虚拟地址中，如 x86 下将PTE 页表基址映射到 0xC0000000，将 PDE 映射到 0xC0300000。Drunkmars 在文章《x64下隐藏可执行内存》指出在 Windows 7 x64 PTE_BASE = fffff68000000000 固定映射，但是从 Windows 10 1607 开始页表采用随机地址映射。

这一块主要涉及两种获取 PTE_BASE 的方法和解析 PTE 的方式：

获取 PTE_BASE 两种常用方法：

  1. 动态获取 PTE_BASE 对应的虚拟地址，可以通过特征码搜索。
  2. 利用页表自映射（Self-Referencing Page Tables）首先获取 PLM4T 的虚拟基地址，然后再获取 PTE_BASE。

解析 PTE 的方式的方式：

PTE 表在物理内存中是一块 4KB 大小的物理页，但是所有物理页面的 PTE 映射到虚拟地址后，这些虚拟地址都是连续的。这就有一个比较有趣的现象，表示 PTE 的虚拟地址是连续的（不止4KB）。

示例1：

  1. 先拿到一个虚拟地址，将其作为基址，如地址 0x0。

    可以看到，线性地址 0x0 对应的 PTE 为 0xFFFF8F8000000000。

  2. 解析地址 0x20DEAC52A40 对应的 PTE。

    可以看到地址 0x20DEAC52A40 对应的 PTE 为 ffff8f81'06f56290。

从上面这个示例可以总结几点：

  • 为什么选定 0x0 的 PTE 作为转换其他线性地址对应的基础地址后，也就是 PTE_BASE？是因为在线性地址中，所有的 PTE 都是从 0x0 对应的 PTE 进行线性排列的。

    • 先右移 12 位，目的就是拿到 PTi 索引。每一项为 8 字节，所以还要左移 3 位。
    • 然后加上基地址 FFFF8F8000000000 即可得到目标地址对应的 PTE。

  • 计算 PTE 的公式：PTE = PTE_BASE + PLM4i*512GB + PDPTi*1GB + PDi*2MB + PTi*8，实际上右移 12 位后低 9 位就是 PTi，左移 3 位就是 *8，只不过使用移位更方便一些。

另外一个疑问，已知一个地址的 PTE（虚拟地址），如何得到该虚拟地址对应的 PDE、PDPTE、PLM4E 对应的虚拟地址。

答案：只需要将当前得到的 PTE 虚拟地址当做一个普通地址，然后解析该地址的 PTE。因为新的 PTE 指向的物理地址保存的内容就是原来的 PTE。

地址 0xffff8fc7'c0837ab0 即为 PDE，物理页面保存的 70e29867 即为 PTE 物理地址。3d28a867 为虚拟地址 0x20DEAC52A40 对应的物理地址。

2 页表自映射

微软管理页表时，使用了页表自映射的技术（Self-Referencing Page Tables，或 Self-ref entry）。

其核心就是：在某一级的页表中存在一个条目，该条目指向当前级页表基址。

  • 32-bit paging、PAE 分页下，该条目位于 PDE 中。
  • 4-level 分页下，该条目位于 PLM4T 中。

注意：Cr3 寄存器中存储了最高级页表（PML4）的表基物理地址。

该自映射的条目位于 PLM4T 中，同时该条目指向 PLM4T 表。因为 CR3 也指向 PLM4T 表，所以：这个 PLM4E entry 物理地址 = CR3，这种自映射也叫做最高级页表自映射。

  1. 推指向 PLM4T 表的虚拟基址。假设在 PLM4T 中，这个自映射页表条目的虚拟地址对应的 4 级分页索引分别为：$x,y,z,r$。页表解析规则：

    当且仅当 $x = y = z = r$ 时，满足上述条件。所以指向 PLM4T 表基址的虚拟地址的特点为：PML4i == PDTi == PDi == PTi == Index。

  2. 推导指向 PDPT 表的虚拟基址。假设这个虚拟地址对应的 4 级分页索引分别为：$a,b,c,d$。

    注意 ![PTE + d * 8] != Cr3，不再指向 PLM4T 表，而是 d = 0。

经过最后的推导，所有指向每级页表的虚拟地址有如下特征：

  • 指向 PML4 表的虚拟地址：PML4i == PDTi == PDi == PTi == Index，页内偏移均为0。
  • 指向 PDPT 表的虚拟地址：PML4i == PDTi == PDi == Index && PTi == 0，页内偏移均为0。
  • 指向 PDT 表的虚拟地址：PML4i == PDTi == Index && PDi == 0 && PTi == 0，页内偏移均为0。
  • 指向 PTT 表的虚拟地址：PML4i == Index && PDTi == 0 && PDi == 0 && PTi == 0，页内偏移均为0。

这个 Index 就是自映射表项在 PML4 表中的索引。
